1. System Overview

Beatly is a data-driven cardiovascular assessment engine that generates personalized heart health insights grounded in real medical thresholds from WHO, AHA/ACC, ADA, and ATP-III published standards.

Core Design Principles

PrincipleImplementation
Data-BackedEvery diagnostic conclusion maps to a published clinical guideline threshold
AdaptiveQuestions change dynamically based on previous answers
PersonalizedSmall input changes produce meaningfully different outputs
TransparentConfidence scores, citations, and causal explanations for every finding
ResilientFull offline fallback engine mirrors backend logic client-side

What the System Does NOT Do

  • ❌ Give generic or template-based advice
  • ❌ Repeat the same outputs regardless of input
  • ❌ Ask fixed/static questions
  • ❌ Generate conclusions without data backing
  • ❌ Diagnose medical conditions (awareness tool only)

💡 How it works: The questionEngine.js module calls buildQuestionPath(form) after every answer. It concatenates the base questions with conditional blocks triggered by the user's current answers. The path is deduplicated by question ID, and getNextQuestion(form) returns the first unanswered question in the built path.

9. Risk Scoring Engine

9.1 Four-Component Scoring

The final risk score is a weighted combination of four independent scoring systems:

ComponentSourceWhat It Measures
ML Score Stacked ensemble (XGBoost, LightGBM, CatBoost, RF, LR) Pattern-based prediction from training data
Rule Score Clinical threshold logic Deterministic scoring from medical standards
Lifestyle Score Points-based system Activity, sitting, stress, sleep, diet, exertion
Personalization Score Cross-signal pattern detection Compound risk from intersecting signals

9.2 Dynamic Weight Fusion

Component weights shift based on assessment confidence (determined by how much clinical data is available):

if confidence ≥ 70%:   ML=0.42  Rule=0.30  Life=0.16  Pers=0.12
if confidence 45–69%:  ML=0.38  Rule=0.28  Life=0.20  Pers=0.14
if confidence <45%:    ML=0.34  Rule=0.22  Life=0.28  Pers=0.16

risk_score = wML × ml + wRule × rule + wLife × life + wPers × personalization
risk_score = clamp(0, 100, round(risk_score))

9.3 Risk Level Classification

Score RangeLevelAction Guidance
0–4LowMaintain current patterns, preventive checks
5–9MildMinor adjustments, monitor trends
10–19ModerateTargeted lifestyle changes, doctor check-in
20–29HighPlan doctor visit, structured intervention
30+Very HighPrompt medical consultation

9.4 Derived Metrics

  • Heart Age = max(actual_age, actual_age + (risk_score − 24) × 0.55)
  • Heart Health Score = 100 − risk × 1.45 ± lifestyle bonuses/penalties (range: 18–96)
  • 6-Month Event Risk = 0.6 + risk × 0.12 + symptom_weight × 0.45
  • 3-Year Event Risk = 4 + risk × 0.72 + symptom_weight × 1.2

12. API Reference

POST /api/predict

Primary assessment endpoint. Accepts user data and returns a full personalized assessment.

Request Body

{
  "consent": true,
  "name": "John",              // optional
  "age": 55,
  "gender": "male",            // male | female | other
  "smoking": "yes",            // yes | no
  "diabetes": "no",            // yes | no
  "family_history": "yes",     // yes | no
  "activity_level": "low",     // low | medium | high
  "sitting": "high",           // low | medium | high
  "stress": "high",            // low | medium | high
  "sleep_quality": "poor",     // poor | ok | good
  "diet": "frequent",          // rare | moderate | frequent
  "exertion_response": "strained",  // easy | noticeable | strained
  "systolic_bp": 148,          // optional, 80–240
  "diastolic_bp": 92,          // optional, 40–160
  "cholesterol": 245,          // optional, 90–400
  "hdl_cholesterol": 38,       // optional, 15–100
  "ldl_cholesterol": null,     // optional, 40–300
  "fasting_glucose": 135,      // optional, 50–400
  "triglycerides": null         // optional, 30–600
}

POST /api/refine

Refine an existing assessment with follow-up symptom answers.

Request Body

{
  "payload": { /* same as /predict */ },
  "answers": {
    "chest_tightness": "yes",
    "breathless_easy": "no",
    "palpitations": "no"
  }
}

POST /api/simulate

What-if scenario: simulate how lifestyle changes would affect risk score.

GET /api/history?limit=8

Retrieve recent assessment history from SQLite database.

GET /api/health

Server health check and model status.

13. Offline Fallback Engine

When the backend is unavailable, fallbackEngine.js provides a complete client-side assessment that mirrors the backend's logic:

  • 4-component scoring — ML approximation, rule engine, lifestyle scorer, personalization
  • Clinical interpretation — Same AHA/ADA/WHO thresholds as backend
  • 12 cross-signal patterns — Identical pattern detection rules
  • Primary/secondary issue identification — Same severity-sorted logic
  • Full narrative generation — Daily story, body signals, food guidance
  • Follow-up refinement — Symptom-based score adjustment
  • Local history — Stored in localStorage with 8-assessment cap
✅ Resilience guarantee: The frontend will NEVER show an error to the user. If the API call fails, it transparently falls back to the client-side engine and produces a complete assessment.
